Створює HTML-сторінки документації API з вихідних файлів Java.

Конспект
javadoc {пакети|джерельні файли} [параметри] [@argfiles]

пакети
Назви пакетів, які ви хочете документувати, розділені пробілами, наприклад java.lang java.lang.reflect java.awt. Якщо ви хочете також задокументувати підпакунки, скористайтеся опцією -subpackages, щоб указати пакунки.

За замовчуванням javadoc шукає вказані пакети в поточному каталозі та підкаталогах. Використовуйте параметр -sourcepath, щоб вказати список каталогів, де шукати пакунки.

вихідні файли
Імена вихідних файлів Java, які ви хочете документувати, розділені пробілами, наприклад Class.java Object.java Button.java. За замовчуванням javadoc шукає вказані класи в поточному каталозі. Однак ви можете вказати повний шлях до файлу класу та використовувати символи підстановки, наприклад /home/src/java/awt/Graphics*.java. Ви також можете вказати шлях відносно поточного каталогу.

параметри
Параметри командного рядка, розділені пробілами. Дивіться параметри.

@argfiles
Імена файлів, які містять список параметрів команди javadoc, імена пакетів і імена вихідних файлів у будь-якому порядку.

опис
Команда javadoc аналізує оголошення та коментарі документації в наборі вихідних файлів Java і створює відповідний набір HTML-сторінок, які описують (за замовчуванням) публічні та захищені класи, вкладені класи (але не анонімні внутрішні класи), інтерфейси, конструктори, методи та поля. Ви можете використовувати команду javadoc для створення документації API або документації щодо впровадження для набору вихідних файлів.

Ви можете запустити команду javadoc для цілих пакетів, окремих вихідних файлів або обох. Під час документування цілих пакетів ви можете використовувати опцію -subpackages для рекурсивного проходження каталогу та його підкаталогів або передавати явний список імен пакетів. Коли ви документуєте окремі вихідні файли, передайте список імен вихідних файлів Java. Дивіться прості приклади.

Відповідність
Стандартний документ не перевіряє вміст коментарів документації на відповідність і не намагається виправити будь-які помилки в коментарях документації. Майте на увазі проблеми, які можуть виникнути під час генерування невідповідного виводу або виведення, що містить виконуваний вміст, наприклад JavaScript, під час виконання команди javadoc. Стандартний документ містить опцію -Xdoclint, щоб допомогти вам виявити типові проблеми в коментарях до документації; однак також рекомендується перевірити згенерований вихід за допомогою будь-яких відповідних інструментів відповідності та перевірки.

Щоб отримати додаткові відомості про вимоги відповідності для документів HTML5, перегляньте вимоги відповідності в специфікації HTML5. Щоб отримати додаткові відомості про проблеми безпеки, пов’язані з веб-сторінками, перегляньте Open Web Application Security Project (OWASP).

Обробка вихідних файлів
Команда javadoc обробляє файли, які закінчуються вихідним текстом, та інші файли, описані в розділі «Вихідні файли». Якщо ви запустите команду javadoc, передавши окремі імена вихідних файлів, ви зможете точно визначити, які вихідні файли обробляються. Однак більшість розробників хочуть працювати не так, оскільки простіше передати імена пакетів. Команду javadoc можна запустити трьома способами без явного вказівки імен вихідних файлів. Ви можете передавати назви пакетів, використовувати параметр -subpackages або використовувати символи підстановки з іменами вихідних файлів.
